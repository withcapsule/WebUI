Changelog

A record of what's been shipped. For what's coming next, see the roadmap →

2025

Android app

Full-featured Android client with upload, download, and client-side encryption. Available via GitHub releases ↗. Feature parity with the web UI including the full encryption workflow.

Open source server (AGPL-3.0)

The Capsule server source code was published on GitHub ↗ under the AGPL-3.0 license. Every line that handles your uploads is auditable.

Self-hosting support

Single-binary deployment with SQLite and no external dependencies. Point the CLI at any Capsule server instance. See the self-hosting guide ↗.

CLI tool — Mac, Linux, Windows

Distributable via Homebrew, Cargo, COPR (Fedora), and a shell installer. Supports encrypted transfers, local transfer history, and pointing at a self-hosted server. All server operations available from the terminal.

Cross-client encryption interoperability

Files encrypted on the web can be decrypted in the CLI, and vice versa. The web UI, CLI, and Android app all produce compatible age v1 ciphertext. No proprietary format, no vendor lock-in.

2024

Client-side encryption

The web UI gained an Encrypt checkbox. Files are encrypted entirely in the browser using the age v1 format before upload. The server receives only ciphertext. A 12-word BIP39 mnemonic passphrase is generated locally and never transmitted. The CLI gained an upload-encrypted subcommand with the same standard.

Human-readable three-word file IDs

File IDs changed from random hashes to three lowercase words joined by hyphens, like made-by-sean. Easier to read aloud, type, and remember. The Receive tab gained spacebar-to-hyphen so you can type IDs without reaching for the hyphen key.

QR code on upload

Every successful upload now generates a QR code alongside the file ID. Scan it directly from the screen to open the download link on a phone.

Web upload & download

Initial launch. Upload any file from the browser, get a short link and QR code back instantly. No login, no account. Files auto-delete after 60 minutes.

curl upload

The API was designed curl-first from the start. One command, one link. 

curl -F "f=@file.txt" https://send.withcapsule.dev/upload

Bandwidth limiting

2 GB per hour per uploader to keep the service free and fair for everyone. Enforced at the server, not the edge.
